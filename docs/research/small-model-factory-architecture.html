Provider-Neutral Stepwise Harness

1 model loaded · tiny steps larger chunks · same gates

1. Step Plan

Break the story into small, resumable steps with acceptance criteria.

qwen2.5-coder:14b can own this. A larger model can plan bigger slices.

2. Context Pack

Deterministic search selects only relevant excerpts, tests, and commands.

No full-repo prompt dump.

3. Patch Proposal

Model returns schema-bound JSON plus a unified diff or search/replace edit.

Malformed output gets one repair.

4. Verify

Harness applies the patch, runs the cheapest relevant check, and records evidence.

Failures become observations.

5. Repair / Next

One bounded repair, then next step, shrink, or escalate to a bigger runtime.

Progress is resumable overnight.

Output Contract

Escalation

Malformed twice, failing twice, too much scope, or missing context → shrink or upgrade.

Overnight Mode

Run local-small with one loaded model, persistent step state, cheap checks after every patch, and no auto-merge. The goal is steady progress without cloud spend, not maximum token speed.

Active Work Mode

Switch the same factory to workhorse models for larger chunks. The harness stays identical: only the profile changes chunk size, runtime, and escalation thresholds.
